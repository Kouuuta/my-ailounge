# Context Engineering

## Overview

Context Engineering is the practice of designing, organizing, and maintaining information in a way that enables both humans and AI systems to understand, navigate, and contribute effectively to a project.

While Prompt Engineering focuses on crafting individual prompts, Context Engineering focuses on the broader ecosystem of information that surrounds those prompts. This includes repository structure, documentation, instructions, research notes, diagrams, standards, and contributor knowledge.

In modern AI-assisted development workflows, Context Engineering is often more important than Prompt Engineering because the quality of an AI's output is heavily influenced by the quality and availability of contextual information.

---

## What is Context Engineering?

Context Engineering is the process of providing AI systems with the information necessary to make accurate, relevant, and consistent decisions.

Examples of context include:

* Project documentation
* Repository structure
* Coding standards
* Architecture diagrams
* Research notes
* Historical decisions
* Contributor instructions
* Task requirements

The goal is to reduce ambiguity and ensure that both humans and AI agents operate with the same understanding of the project.

---

## Why Context Engineering Matters

Without proper context, AI systems may:

* Generate inconsistent solutions
* Duplicate existing work
* Misinterpret requirements
* Ignore established standards
* Produce outputs that do not align with project goals

With proper context, AI systems can:

* Produce more accurate outputs
* Follow project conventions
* Understand repository structure
* Assist with documentation and research
* Contribute more effectively to development workflows

---

## Context Engineering vs Prompt Engineering

| Prompt Engineering              | Context Engineering                           |
| ------------------------------- | --------------------------------------------- |
| Focuses on individual prompts   | Focuses on the entire information environment |
| Short-term guidance             | Long-term knowledge structure                 |
| Task-specific                   | Repository-wide                               |
| Influences a single interaction | Influences every interaction                  |

Prompt Engineering helps AI answer a question.

Context Engineering helps AI understand the project.

## Sources of Context

### Repository Structure

A well-organized repository provides context by showing where information belongs and how different resources relate to one another.

### Documentation

Documentation explains project goals, workflows, standards, and technical details.

### Research Notes

Research documents preserve findings and decisions that can be referenced later.

### Diagrams

Visual representations provide architectural and system-level context.

### Contributor Workspaces

Personal logs, TODO lists, and research notes help preserve institutional knowledge.

### Instruction Files

Instruction documents guide contributors and AI tools on how to interact with the repository.

## Best Practices

### Maintain Clear Documentation

Documentation should be updated whenever significant changes are introduced.

### Organize Information Logically

Files should be grouped according to purpose and responsibility.

### Preserve Research

Research findings should be documented rather than stored solely in conversations or personal notes.

### Record Decisions

Important decisions should be documented to provide historical context.

### Reduce Ambiguity

Folder names, file names, and documentation should clearly communicate intent.

### Design for AI Readability

Documentation should be structured, descriptive, and easy for AI systems to parse and understand.
